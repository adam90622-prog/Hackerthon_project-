# /strategize — 발행 전략안 생성 (Challenge)

## 목적
Standard 산출물(파이프라인 실행 결과·발행 캘린더)에서 드러난 패턴을 근거 데이터로 삼아,
문제 정의/근거 데이터/전략 제안/우선순위 4단 구성의 발행 전략안을 생성합니다.

## 전제 조건
1. /analyze, /insight, /generate 완료 (Basic)
2. `scripts/run_pipeline.py` 실행 결과(타깃별 k값, 캐던스 PASS/FAIL) 확보 (Standard)
3. `output/standard-guide.md`의 캘린더(ref_event 분포) 확인

## 절차
1. `scripts/check_cadence.py` 결과에서 타깃별 k값·FAIL 여부를 추출해 상시 콘텐츠
   부족 문제의 근거로 사용
2. 정제된 CSV에서 ref_event가 채워진 행의 비율·이벤트 유형을 집계해 시의성 대응
   문제의 근거로 사용
3. 위 두 인사이트를 아래 우선순위 기준으로 각각 채점

## 우선순위 판단 기준
영향범위(0~2, 몇 개 세그먼트에 영향) + 시급성(0~2, 방치 시 손실 회복 가능성) +
실행용이성(0~2, 적은 리소스로 실행 가능한가)을 합산합니다.
- 5~6점 → P0(즉시 착수)
- 3~4점 → P1(다음 사이클 착수)
- 0~2점 → P2(장기 검토)

## 출력 형식
```
## 전략안 N: 제목
**문제 정의**
**근거 데이터**
**전략 제안**
**우선순위**
```

## 완료 조건
- 전략안 최소 1~2건, 각 근거 데이터가 실제 스크립트/CSV 결과 수치에 연결됨
- 우선순위 점수·등급이 명시됨
- `output/challenge-strategy.md`에 저장됨
