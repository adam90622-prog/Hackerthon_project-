# /generate — SNS 콘텐츠 3종 세트 생성

## 목적

data/nursevillage_content_seeds.csv의 각 행에 대해 카드뉴스·카카오메시지·뉴스레터 3종을 생성하고
output/output_content_set.md에 저장합니다.

---

## 전제 조건
1. /analyze 완료 — 결측·중복 행 파악됨
2. /insight 완료 — 채널·용어·시의성 전략 수립됨
3. `context/company-info.md`(용어 사전) 읽기 완료
4. `context/industry-news.md`(시의성) 읽기 완료

---

## 생성 규칙

### 채널별 형식·분량 (필수)
| 채널 | 기준 | 측정 |
|---|---|---|
| 카드뉴스 | 3~5장, 장별 헤드라인 + 본문 2~3문장, 마지막 장 CTA | 장 수·구성 확인 |
| 카카오메시지 | 200자 이내, 핵심 + 링크 유도 | 공백 포함 글자수 세기 |
| 뉴스레터 | 600자 내외, 도입–본문–CTA 3단 | 공백 포함 글자수 세기 |

### 톤 변형 규칙
각 행의 `tone`(공감형/정보형/유머형)을 3채널에 적용하되 채널 특성에 맞게 변형:
- 카드뉴스: 톤을 스토리텔링 구조로
- 카카오: 톤을 간결한 행동 유도형으로
- 뉴스레터: 톤을 정보 깊이 있는 3단 구조로

### 현장 용어 처리 (필수)
- keywords_required의 용어를 **전 채널에 자연 삽입**
- 각 행 끝에 **사용된 현장 용어 목록**을 태그 형식으로 (예: `#프리셉터 #태움 #SBAR`)

### 결측 keywords 처리 (id 12)
- topic·target·tone 기반으로 적절한 현장 용어를 추론·보완
- 보완값은 대괄호 표시: `[추론: 보완 용어]`, 교정 내역은 decisions.md 기록

### ref_event 처리
- 값이 있으면 도입·CTA에 자연 연계 / 빈칸이면 시즌 언급 없이 상시 콘텐츠로

### 중복 topic 처리 (id 11·17)
- 같은 제목이라도 target별로 사례·어휘·CTA를 다르게 작성

---

## 생성 순서
1. id 01부터 순서대로 처리
2. 각 행마다 아래 출력 형식으로 3종 생성
3. 생성 후 카카오·뉴스레터 글자수를 실제로 세어 기재
4. 18행 완료 후 output/output_content_set.md에 전체 저장

---

## 출력 형식 (output/template.md 참고)
```markdown
## 시드 #[id]: [topic]
**타깃**: [target_audience] | **톤**: [tone] | **시의성**: [ref_event 또는 없음]

### 🎴 카드뉴스 ([N]장)
1장 [헤드라인] / 본문…
2장 [헤드라인] / 본문…
…
[마지막 장] [CTA]

### 💬 카카오메시지 (___자)
[200자 이내 — 핵심 + 링크 유도]

### 📰 뉴스레터 (___자)
[도입] …
[본문] …
[CTA] …

### 🏷️ 사용된 현장 용어
#용어1 #용어2 #용어3

---
```
