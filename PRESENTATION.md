# Problem 04 · 다이버즈 — SNS 콘텐츠 3종 동시 자동 작성

## 1. 배경
다이버즈(Divers)는 60만 간호사·간호학생의 커리어 플랫폼 **널스빌리지(NurseVillage)**를
운영하는 헬스케어 HR 기업입니다. 널스빌리지는 간호사·간호학생을 대상으로 SNS 콘텐츠
(카드뉴스·카카오메시지·뉴스레터)를 정기 발행하며, **1명의 마케터가 제작을 전담**하고 있습니다.

## 2. 문제
1) **시간 효율**: 1건 발행에 2~3시간이 걸려, 발행 빈도와 대응 속도에 한계가 있습니다.
2) **품질 일관성**: 담당자가 바뀌면 브랜드 보이스가 흔들리고, 현장 용어(태움·프리셉터·SBAR 등)가
   빠지면 독자 공감도가 떨어집니다.

## 3. 해결 방향

**문제 1) → 재사용 가능한 생성 파이프라인**
데이터 정제(결측·중복·이상치 처리) → 3종 콘텐츠 동시 생성 → 글자수·용어 자동 검증까지,
반복되는 작업을 스크립트 + Claude Code 대화로 자동화했습니다. 새 시드 데이터(50행 더미)를
넣어도 같은 로직이 동일하게 재현되는지 직접 검증했습니다.

**문제 2) → 문서화된 브랜드보이스 + 데이터 기반 발행 전략**
실제 널스빌리지 앱·아티클을 분석해 호칭·어투·이모지 사용 규칙을 가이드로 문서화하고,
현장 용어 반영을 검증 단계에 포함시켜 담당자가 바뀌어도 톤이 유지되게 했습니다. 발행 시점도
"감"이 아니라 이벤트 날짜·리텐션 캐덴스(14일 기준) 근거로 설계했습니다.

## 4. 결과물

### Basic
- 시드 18건 전체 카드뉴스(4장)·카카오메시지(≤200자)·뉴스레터(550~650자) 생성
- 현장 용어 태그, 타깃·톤별 채널 변형 완료, 18/18 글자수·용어반영 검증 PASS
- 결측치(id12)·중복 topic(id11=17, 관점 분리) 정제 완료

### Standard
- 재사용 파이프라인: clean_seeds.py(정제) + run_pipeline.py(정제+캐덴스 검증) —
  50행 더미 데이터로 재현성 검증(바이트 단위 동일 결과 확인)
- 발행 캘린더: 시의성 이벤트 직전 발행 원칙 + 세그먼트별 14일 캐덴스 규칙 + 확정구간(14일)
  보호 로직
- 채널별 핵심 지표: 카드뉴스=저장률, 카카오=클릭률, 뉴스레터=오픈율(+CTOR)
- 브랜드보이스 가이드: 실제 앱 분석 기반 호칭·이모지·인용 규칙 문서화
- 콘텐츠 생명주기 메타데이터(evergreen/recurring_seasonal/time_bound_fact) 설계

## 5. 과정에서의 시행착오 (self-review)
- id12 결측 키워드 추론값이 재실행마다 달라지는 재현성 문제를 발견 → 최종 수동 확정 후
  고정, decisions.md에 근거 기록
- 캐덴스 계산(70÷k) 결과가 원본 18건 데이터만으로는 구조적으로 FAIL — 데이터 볼륨이
  원인임을 50행 더미로 검증해 확인
- 실제 앱 브랜드보이스(호칭·이모지)와 초안 콘텐츠의 불일치를 발견해 18건 전체 재작업

## 6. 제외 범위
카드뉴스 이미지 생성, 실제 SNS 자동 발행, Challenge(발행 전략안 자동 생성)는 시간상
이번 라운드에서 진행하지 못했습니다.
