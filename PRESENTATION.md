# Problem 04 · 다이버즈 — SNS 콘텐츠 3종 동시 자동 작성

## 1. 배경
다이버즈(Divers)는 60만 간호사·간호학생의 커리어 플랫폼 **널스빌리지(NurseVillage)**를
운영하는 헬스케어 HR 기업입니다. 널스빌리지는 간호사·간호학생 대상 SNS 콘텐츠(카드뉴스·
카카오메시지·뉴스레터)를 정기 발행하며, 마케터 1명이 제작을 전담합니다.

## 2. 문제
1) **속도**: 3종 포맷을 각각 새로 써서 1건당 2~3시간 소요
2) **일관성**: 담당자가 바뀌면 브랜드 보이스(호칭·이모지)와 현장 용어 반영이 흔들림
3) **시의성 대응**: 기획→초안→수정 사이클이 길어 국시 합격발표·간호법 같은 이벤트
   대응이 늦음

## 3. 해결 방향
데이터 정제(결측·중복 처리) → 3종 콘텐츠 동시 생성 → 캘린더·지표·브랜드보이스 설계 →
전략안 도출까지, Notion 스펙이 요구한 Basic→Standard→Challenge 범위 그대로 구현했습니다.
1차 시도에서 스펙 밖 기능(대시보드 UI, 신규 기사 자동분류, 발행 이력 시뮬레이션·재스케줄링,
합성 더미 데이터)을 과도하게 만들었던 것을 재점검해 걷어내고, 스펙이 명시한 산출물에
집중하는 방향으로 재구성했습니다.

## 4. 결과물

### Basic
- 18개 시드 전체 카드뉴스(4장)·카카오메시지(≤200자)·뉴스레터(550~650자) 생성
- 현장 용어 태그, 타깃·톤별 채널 변형 완료
- id 12(키워드 결측)·id 11/17(중복 topic) 함정 데이터 처리 완료

### Standard
- 재사용 파이프라인: `clean_seeds.py`(정제) → `check_cadence.py`(캐던스 점검) →
  `run_pipeline.py`(래퍼) → `validate_content.py`(생성물 검증)
- 발행 캘린더(원본 18건, 시의성 이벤트 D-1~D-30 역산 배치 + 상시 콘텐츠 생애주기 배분)
- 채널별 핵심 지표: 카드뉴스=저장률, 카카오=클릭률, 뉴스레터=오픈율+CTOR
- 브랜드보이스 가이드: 호칭 병기·감성 이모지 팔레트·익명 집단 화법 3원칙

### Challenge
- 전략안 1(P0): 세그먼트별 상시 콘텐츠 확충 — `check_cadence.py` 실측 k값(3~4개)이
  목표 캐던스(14일) 충족에 필요한 k=5에 못 미침을 근거로 제시
- 전략안 2(P1): 시의성 이벤트 리드타임 표 고정 + 연간 갱신 루틴 — ref_event 8/18건(44%)
  분포를 근거로, 국시 등 매년 변동되는 날짜를 "규칙 고정·날짜만 갱신"하는 구조로 제안
- `/strategize` Skill로 패키징(재현 가능)

### KPI (문제 해결도 실측)
| 지표 | 결과 |
|---|---|
| 생성 성공률 | 카드뉴스 18/18 · 카카오 18/18 · 뉴스레터 18/18 (100%) |
| 브랜드보이스 유지도 | 100% (호칭·이모지·카카오 무이모지 규칙 3종 전부) |
| 전문용어 사용도 | 100% (keywords_required 55/55 반영) |
| 속도 단축률 | 96.7% (건당 150분→5분, 18건 기준 45시간→1.5시간) |

## 6. 제외 범위
카드뉴스 이미지 생성, 실제 SNS 자동 발행, 영상/릴스, 콘텐츠 성과분석, 의료 팩트체크,
실제 운영 데이터는 Notion 스펙대로 제외했습니다. 추가로, 대시보드/웹 UI 신규 구현·
신규 기사 자동분류·발행 이력 시뮬레이션 및 재스케줄링·합성 더미 데이터 생성은 스펙 범위를

`dashboard.html`은 이번에 콘텐츠+KPI 리포트로 교체했습니다).
