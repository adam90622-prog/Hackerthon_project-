1. 문제 재정의

"카드뉴스를 많이, 빨리 만든다"가 문제였던 게 아니라 —

"타깃(신규RN·간호학생·경력RN)별 상시 콘텐츠 개수(k)가 구조적으로 부족해서, 발행 순서를 아무리 잘 짜도 70÷k 계산상 14일 캐던스를 만족시킬 수 없다"는 것이 실제 문제였습니다.

원본 18건만으로 scripts/check_cadence.py를 돌리면 3개 타깃 전부 FAIL(23.3 / 23.3 / 17.5일)이 나옵니다. 이게 "배치 로직이 잘못됐다"인지 "데이터가 부족한 것"인지를 가르기 위해 data/dummy_seeds_50_cleaned.csv 50행으로 같은 스크립트를 다시 돌렸고, 이번엔 3개 타깃 전부 PASS(6.4 / 7.0 / 5.4일)가 나왔습니다. 로직은 그대로 두고 입력 볼륨만 늘렸는데 결과가 바뀌었다는 것 — 그래서 "잘 쓰는 것"보다 한 단계 위에 "재사용 가능한 파이프라인 + 타깃당 최소 5개의 상시 소재 확보"라는 문제가 있다는 걸 수치로 확인했습니다.

D-01

결측 키워드 추론값이 재실행마다 달라짐 (id 12)

문제
id 12("국시 끝나고 우리가 하고 싶은 것들")의 keywords_required가 결측이라 규칙 기반으로 채워야 했는데, 같은 규칙을 다시 돌릴 때마다 채택되는 후보가 바뀌는 재현성 문제가 드러났습니다.
판단
ref_event가 가리키는 이벤트를 "시행일 vs 발표일" 두 국면으로 나누고 topic이 어느 국면에 속하는지부터 다시 따지는 국면 기반 재판단을 시도했지만, 이 재판단 과정 자체를 반복해도 값이 계속 흔들리는 근본 문제는 남았습니다.
선택
더 이상 규칙 재판단을 반복하지 않고 "국시,면허,RN"으로 값을 직접 확정한 뒤, clean_seeds.py의 MANUAL_OVERRIDES["12"]에 고정 — 재실행해도 항상 같은 값이 나오도록 만들었습니다.
근거: decisions.md — "keywords_required 결측 보완 — id 12" ~ "keywords_required 최종 확정(재판단 중단) — id 12"
D-02

14일 캐던스가 원본 18건만으로는 구조적으로 FAIL

문제
타깃별 최소 14일 캐던스를 목표로 재발행 23건까지 투입했는데도 최악의 공백이 신규RN 44일, 간호학생 54일, 경력RN 70일까지만 줄었을 뿐, 세 타깃 모두 14일 목표를 못 채웠습니다.
판단
이게 배치 로직의 결함인지, 애초에 상시 콘텐츠 개수(k)가 부족한 데이터 문제인지를 구분해야 했습니다. scripts/check_cadence.py로 70÷k 공식을 코드화해, 원본 18건과 50행 더미 데이터에 각각 돌려 비교했습니다.
선택
18건은 k=3~4로 3개 타깃 전부 FAIL(23.3~17.5일), 50행 더미는 k=10~13으로 전부 PASS(5.4~7.0일) — 같은 로직에 입력 볼륨만 다르게 넣었더니 결과가 갈린다는 걸 확인해, 원인이 로직이 아니라 "타깃당 상시 콘텐츠 최소 5개 필요"라는 데이터 볼륨 문제임을 결론지었습니다.
근거: decisions.md — "세그먼트별 캐던스(14일) 및 재발행 규칙 도입" · "18행 vs 50행 캐던스 재검증"
D-03

실제 앱 브랜드보이스와 초안의 불일치

문제
널스빌리지 실제 앱·아티클을 분석해보니 호칭·이모지 사용 규칙이 이미 만들어둔 18건 초안과 어긋나 있었고, 증언·인터뷰처럼 보이는 인용을 넣을지도 정해야 했습니다.
판단
특정 인물에게 귀속되는 인용("~병원 n년차")을 넣는 방안을 검토했지만, 정제된 시드 데이터에는 실제 증언·인터뷰 원본이 없어 이를 채우면 지어낸 인용이 된다는 문제를 확인했습니다.
선택
호칭("선생님" 세트당 최소 1회)·이모지(톤별 팔레트)·인용(가상 인물 대신 "선배들의 이야기를 들어보면" 익명 집단 화법)이라는 3원칙을 context/nursevillage-brand-voice.md로 문서화하고 18건 전체를 재작업, 글자수도 재검증(2건이 상한을 넘어 재조정)했습니다.
근거: decisions.md — "브랜드보이스 3종 규칙 정합화 — 18행 전체"
D-04

"오늘" 기준으로만 과거를 고정하면 이미 확정된 일정도 뒤집힘

문제
재스케줄링 로직이 "TODAY 이전"만 고정 구간으로 보면, 카피 작성·검수·예약 발행이 이미 끝나 있을 법한 내일모레치 콘텐츠까지 재배치 대상이 되어버리는 문제가 있었습니다.
판단
고정 구간을 며칠로 늘릴지(7일 단위 vs 14일 단위)를 검토했는데, 이 과제의 캐던스 목표 자체가 14일이라 확정 구간을 다른 길이로 잡으면 "확정 구간 하나가 캐던스 목표를 못 채우는" 혼란이 생긴다는 문제를 발견했습니다.
선택
TODAY~TODAY+14일을 "확정 구간"으로 추가해 과거/확정/자유 구간 3분할로 바꾸고, 그 구간 안에서도 실제 이벤트 날짜가 있고 최소 3일 리드타임이 확보되고 빈 슬롯일 때만 예외적으로 시의성 콘텐츠를 끼워 넣는 규칙(scripts/reschedule.py)까지 설계했습니다.
근거: decisions.md — "확정 구간(committed window) 규칙 도입" · "확정 구간 내 신규 시의성 콘텐츠 예외 삽입 규칙"
D-05

같은 topic 문자열이 서로 다른 타깃 행에 중복 등장

문제
"실습 나가서 듣는 웃픈 한마디 모음"(간호학생 id11 / 경력RN id17), "선배가 알려준 인계 꿀팁 모음"(신규RN id48 / 경력RN id49)처럼 topic 문자열이 완전히 같은 행이 서로 다른 타깃에 중복으로 존재했습니다.
판단
topic 문자열 자체를 타깃별로 다르게 바꿔 쓰는 방안을 검토했지만, 원문 topic을 임의로 고치는 건 정제 단계의 역할을 넘어선다고 보아 보류했습니다.
선택
원문 topic은 그대로 두고 DUP01로 플래그만 남긴 뒤, 관점만 분리(예: 간호학생 = '당하는' 실습생 1인칭, 경력RN = '돌아보는' 선배 시점)하도록 판단 근거를 다음 콘텐츠 생성 단계로 넘겼습니다.
근거: decisions.md — "중복 topic 플래그 — DUP01" (id 48/49, id 11/17 각 1건씩 총 2건)
