# 기획안 — 다이버즈 SNS 콘텐츠 3종 동시 자동 작성

> 이 문서는 기존 `PRESENTATION.md`를 대체합니다. 1차 산출물이 문제 스펙(Notion 원문)
> 범위를 벗어났다는 진단에 따라, **요구사항에 정확히 맞춰 범위를 재설정**하고 이후
> 작업(Basic → Standard → Challenge → KPI 측정 → HTML 리포트)이 이 계획을 따르도록

## 1. 문제 재정의 (기준: Notion 원문)

널스빌리지는 간호사·간호학생 대상 SNS 콘텐츠(카드뉴스·카카오메시지·뉴스레터)를 마케터
1명이 전담 제작합니다. 핵심 병목은 세 가지입니다.

1. **속도**: 3종 포맷을 각각 새로 써서 1건당 2~3시간 소요
2. **일관성**: 담당자가 바뀌면 브랜드 보이스(호칭·이모지)와 현장 용어 반영이 흔들림
3. **시의성 대응**: 기획→초안→수정 사이클이 길어 국시 발표·간호법 같은 이벤트 대응이 늦음

해결해야 할 것은 "새로운 기능"이 아니라 **이 세 가지 병목을 없애는 3종 콘텐츠 생성·운영
방식**입니다. 데이터 분류 시스템이나 재발행 스케줄러 같은 부가 기능은 문제가 요구하지
않았고, 오히려 세 병목과 무관한 별도 과제가 됩니다.

## 3. 레벨별 목표 재정의

### 🟢 Basic — 콘텐츠 마케터 수준
18개 시드 각각에 대해 카드뉴스(3~5장)·카카오메시지(200자 이내)·뉴스레터(600자 내외) 3종을
생성하고, 사용된 현장 용어를 태그로 정리한다. **산출물은 스펙이 지정한 항목 그대로만
포함**한다(3종 콘텐츠 세트 + 용어 태그 + 글자수 준수 — 그 이상도 이하도 아님).

### 🟡 Standard — 마케팅 PM 수준
Basic을 반복 가능한 파이프라인으로 만든다.
- **재현 파이프라인**: 정제 → 전략 → 3종 생성이 새 seeds CSV에도 동일 품질로 도는
  스크립트/Skill + 짧은 사용법. 검증은 **제공된 데이터 범위 안에서**만 한다(신규 데이터
  생성 금지 — 제외범위 "실제 운영 데이터 사용" 원칙과 같은 취지로, 스펙에 없는 대량
  더미 생성도 지양).
- **콘텐츠 캘린더 + 채널별 지표 + 브랜드보이스 가이드**: "마케팅·운영 관점에서 무엇이
  유용한가"를 기준으로 직접 설계하되, **근거는 1~2줄 요약으로 충분**하다. 캐던스
  계산(70÷k) 같은 정량적 통찰은 결론만 담고, 알고리즘 구현은 하지 않는다.

### 🔴 Challenge — 콘텐츠 전략가 수준
Standard 산출물 전체 + seeds 패턴(타깃·톤·시의성)에서 도출한 발행 전략안 1~2건. 각
전략안은 문제 정의 / 근거 데이터 / 전략 제안 / 우선순위 4단 구성을 따르며, 형식과
우선순위 판단 기준은 직접 설계한다.

- **전략안 1**: 세그먼트별 상시(evergreen) 콘텐츠 부족 → 14일 캐던스 구조적 불가 (Standard
  캘린더 작업에서 나온 k값 인사이트를 근거 데이터로 활용)
- **전략안 2**: `ref_event` 기반 시의성 대응 전략 (국시·간호법 등 이벤트 앵커링 원칙)

## 4. KPI 설계 (문제 해결도 측정)

이 프로젝트의 목적은 "3종 동시 자동 작성으로 병목이 해소되는 정도"를 보여주는 것이므로,
문제 정의(①속도 ②일관성 ③시의성) 및 스펙 성공 기준(현장 용어 반영, 타깃·톤 일치)에
직접 대응하는 지표만 측정한다.

| KPI | 측정 방법 | 대응하는 문제 |
|---|---|---|
| **속도 단축률** | 기존 수작업 기준(1건 2~3시간) 대비, AI 파이프라인 예상 소요 시간 단축 비율 | 병목 ① 속도 |
| **브랜드보이스 유지도** | 18건 × 3채널에서 호칭 병기·이모지 규칙 준수 비율 | 병목 ② 일관성 |
| **전문용어 사용도** | `keywords_required` 반영률 (지정된 용어가 실제 본문에 등장한 비율) | 문제 스펙 "현장 용어 반영" |
| **생성 성공률** | 채널별 형식·글자수 조건 충족 건수 / 18건 (채널별 성공 건수·비율로 표기) | Basic "3종 포맷 정확도" |

측정 대상은 스펙이 지정한 18건 원본 시드로 한정한다(신규/합성 데이터 사용 안 함).

## 6. 작업 순서

기획안(현재 완료) → Basic 검수 → Standard 재작성 → Challenge 전략안 → KPI 측정 →
HTML 리포트 → 회고록 정리. 각 단계는 순차적으로 1개씩 진행하며, 완료할 때마다
회고록의 해당 항목을 채워나간다.

## 7. 이번 라운드에서 지키는 제외 범위

스펙이 명시한 제외 범위(카드뉴스 이미지 생성, 실제 SNS 자동 발행, 영상/릴스, 콘텐츠
성과분석, 의료 팩트체크, 실제 운영 데이터)에 더해, **1차 산출물에서 스스로 만들었던
초과 범위도 동일하게 제외**한다: 대시보드/웹 UI, 신규 기사 분류 시스템, 발행 이력
시뮬레이션 및 미래 재스케줄링 알고리즘, 합성 더미 데이터(50행) 생성.
