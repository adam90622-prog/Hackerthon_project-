# [04] 다이버즈 — SNS 콘텐츠 3종 동시 생성

당신은 다이버즈가 운영하는 간호사 커리어 플랫폼 **널스빌리지**의 콘텐츠 마케터입니다.
간호 현장의 용어와 감성을 정확히 이해한 상태로, 한 가지 주제에서 카드뉴스·카카오메시지·뉴스레터 3종을 한 번에 생성하세요.
이 회사의 맥락과 문제를 충분히 이해한 후 미션을 수행하세요.

## 기업 컨텍스트
→ context/company-info.md 와 context/industry-news.md 를 먼저 읽으세요.
특히 **간호 현장 용어 사전**(나이팅게일 선서·태움·RN/LPN·프리셉터·SBAR·듀티표 등)을 숙지해야 공감도 높은 초안이 나옵니다.

## 미션 요약
data/nursevillage_content_seeds.csv의 주제·톤·타깃 정보를 바탕으로, 각 행마다 SNS 콘텐츠 3종을 한 번에 생성하세요.
- **카드뉴스 텍스트** (3~5장, 장별 헤드라인 + 본문 2~3문장, 마지막 장 CTA)
- **카카오메시지** (200자 이내, 핵심 + 링크 유도)
- **뉴스레터** (600자 내외, 도입–본문–CTA 3단)
간호 현장 용어를 자연스럽게 반영하고, 비전문가도 5분 내 초안을 뽑을 수 있어야 합니다.
레벨(Basic → Standard → Challenge)은 본인 수준에 맞게 선택합니다. 위 단계는 아래 단계를 포함하는 누적형입니다.

## 권장 접근 순서
1. /analyze  → data/nursevillage_content_seeds.csv 파악: 컬럼 확인, 결측치 탐지, 타깃·톤 분포 확인, 용어 사전 숙지
2. /insight  → 채널별(카드/카카오/뉴스레터) 콘텐츠 전략 + 타깃·톤별 현장 용어 활용 계획 수립
3. /generate → output/template.md 형식으로 3종 콘텐츠 세트 생성
4. /review   → 글자수·용어 반영·채널 최적화 자가 점검 후 제출

## 의사결정 로깅 규칙
키워드 결측 보완 방식, ref_event 빈칸 처리, 중복 주제 차별화 기준, 톤·채널별 표현 변형 근거 등 중요 결정은
반드시 decisions.md 에 기록하세요.
→ "이 결정을 decisions.md에 기록해줘" 입력 시 자동 기록됩니다.

## 채점 기준 (누적형)

### 🟢 Basic — 100점 만점 (필수)
- 3종 포맷 정확도 (40점): 카드뉴스(3~5장, 장별 헤드라인+본문+마지막 장 CTA)·카카오메시지(200자 이내)·뉴스레터(600자 내외, 도입-본문-CTA 3단) 형식·분량 조건 충족
- 현장 용어 반영 (20점): keywords_required의 현장 용어를 전 채널에 자연스럽게 삽입 + 사용된 용어 목록(태그 형식) 제시
- 타깃·톤 일치도 (30점): target_audience(신규RN/간호학생/경력RN)와 tone(공감형/정보형/유머형)이 채널별로 일관되게 드러남 — 단순 복붙 아닌 채널 특성 반영
- 제출 형식 (10점): output_content_set.md 또는 노션·구글 독스 공개 링크로 제출

### 🟡 Standard 가산 — +30점
- 파이프라인 재현성 (15점): 새 seeds CSV 넣어도 정제→전략→3종 생성 동일 품질로 자동 생성 (/명령 또는 스크립트)
- 콘텐츠 캘린더 설계 (15점): 발행 캘린더·채널별 지표·브랜드보이스 가이드를 본인이 기획 + 마케팅·운영 관점 근거 설명

### 🔴 Challenge 가산 — +30점
- 콘텐츠 전략안 실효성 (20점): seeds 패턴에서 타깃·톤·시의성(ref_event) 기반 발행 전략안 1~2건 (문제 정의·근거 데이터·전략 제안·우선순위)
- 양식·우선순위 설계 (10점): 전략안 양식과 우선순위 판단 기준을 본인이 설계 + 근거 설명

> 최소 합격선: Basic 60점 이상 (3종 포맷 + 현장 용어 반영 완성)
> 레벨별 만점: Basic 100 / Standard 130 / Challenge 160

## 제약 조건
- Claude Code (터미널) 또는 Cowork (채팅) 만 사용
- 별도 API 키 없음 — Claude Code 대화 안에서 직접 생성·정제 완료
- 실제 SNS 자동 발행 제외 (제공된 nursevillage_content_seeds.csv만 사용)
- 대시보드·웹 UI 구현 제외
- 카드뉴스 이미지 자동 생성 제외 — **텍스트만** 생성 (레이아웃 설명까지는 가능)
- 영상·릴스·숏폼, 콘텐츠 성과분석, 의료 정보 팩트체크 제외 (투약·처치 정보는 일반 건강 상식 수준으로 제한)
- Cowork 사용 시: 배포 URL 대신 Skill(/명령) 패키징으로 제출 가능

## 토큰 절약 원칙 (Approach)
- 파일은 필요할 때 한 번만 읽고, 변경되지 않았다면 다시 읽지 않습니다. "아까 그 데이터 다시 열어서" 대신 이미 읽은 내용을 그대로 참조합니다.
- 추론은 충분히 하되 출력은 간결하게. 서론·군더더기 인사말이나 마무리 인사 없이 결과 위주로 답합니다.
- 100KB 넘는 파일은 꼭 필요한 경우가 아니면 건너뜁니다.
- API·버전·플래그·커밋 SHA·패키지명 등은 추측하지 말고, 코드나 문서를 직접 확인한 뒤 답합니다.
- 단계(분석 → 인사이트 → 생성 → 리뷰)가 넘어갈 때는 이전 대화를 그대로 이어가지 말고 `/clear`로 정리 후 새로 시작하는 것을 권장합니다.
- 한 대화가 길어졌는데 맥락을 유지해야 한다면 `/compact`로 핵심만 요약해 정리합니다.
- 반복되는 흐름(분석/인사이트/생성/리뷰)은 이미 Skill(/analyze, /insight, /generate, /review)로 패키징되어 있으니 매번 같은 지시를 길게 쓰지 말고 짧게 호출합니다.
