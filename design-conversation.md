## 1. 결과물 임팩트 논의

### Q. output이 약하다는 고민 — 구체화 또는 대안 아이디어?

**현황 문제**: "SNS 콘텐츠 3종 생성"이 1회성 초안으로 끝나면 AI 활용 역량(타깃·톤·현장 용어·시의성 동시 처리)이 잘 드러나지 않음

**브레인스토밍 결과 — 세 가지 방향 검토**

| 옵션 | 설명 | 결론 |
|------|------|------|
| A. 3종 동시 생성 스크립트/Skill | seeds CSV → 카드/카카오/뉴스레터 자동 생성 파이프라인 | Standard 레벨로 포함 |
| B. 카드뉴스 이미지 자동 생성 | 텍스트 + 이미지까지 생성 | **제외 범위**로 결정 (노션 ⑥항: 텍스트만) |
| C. 발행 전략안(타깃·톤·시의성) 생성 | seeds 패턴 → 콘텐츠 발행 전략 기획 | Challenge 레벨로 포함 |

## 2. 레벨 구조 설계

### Q. 현재 목표를 레벨 1로 하고 레벨 2, 3을 설정한다면?

**설계 원칙**: "1회성 초안" → "매주 돌아가는 콘텐츠 공장" → "데이터 기반 발행 전략"으로 발전하는 누적형. 콘텐츠 생성형 트랙 페르소나(콘텐츠 마케터 → 마케팅 PM → 콘텐츠 전략가).

### 최종 확정 레벨

```
🟢 Basic (100점) — 콘텐츠 마케터 수준
  → 18개 시드를 카드뉴스(3~5장)·카카오(200자)·뉴스레터(600자) 3종 생성
  → 현장 용어 자연 반영 + 사용 용어 태그 / 타깃·톤 일치
  → 제출: output_content_set.md 또는 공개 링크

🟡 Standard (+30점, 누적 130점) — 마케팅 PM 수준
  → Basic + 새 seeds CSV에도 도는 재현 파이프라인
  → 콘텐츠 캘린더·지표·브랜드보이스 가이드 본인 설계

🔴 Challenge (+30점, 누적 160점) — 콘텐츠 전략가 수준
  → Standard + 타깃·톤·시의성 기반 발행 전략안 자동 생성
  → 전략안 구성: 문제 정의 / 근거 데이터 / 전략 제안 / 우선순위
  → 전략안 양식·우선순위 기준도 본인이 설계
```

## 3. CSV 데이터 재설계

### Q. 노션 문서(Problem 04 v2)와 로컬 파일 정합성 맞추기

**변경 전 컬럼** (구버전 — problem-04-divers/data/content-topics.csv, 13행)
```
id, 날짜, 주제, 세부주제, 톤, 타깃, 참고키워드, 작성자메모
```

**변경 후 컬럼** (노션 v2 기준 — nursevillage_content_seeds.csv, 18행)
```
id, topic, tone, target_audience, keywords_required, platform_hint, ref_event
```

**주요 변경 사항**
- 파일명 `content-topics.csv` → `nursevillage_content_seeds.csv` (노션 ④ 지정)
- **행 수 13 → 18**: target 3종(신규RN/간호학생/경력RN) × tone 3종(공감형/정보형/유머형), 각 조합 2건씩 = 18 (균형 구성)
- `날짜`·`세부주제`·`작성자메모` 제거 (정답·노이즈성 컬럼 정리)
- `톤` 값 재정의: 정보성/공감형/긴박한/유머러스 → **공감형/정보형/유머형 3종**(노션 스펙 고정)
- `타깃` → `target_audience` 값 변경: 신규간호사/경력간호사/전체 → **신규RN/간호학생/경력RN**
- `참고키워드` → `keywords_required` (반드시 포함할 현장 용어)
- `platform_hint` 신규 추가 (card/kakao/newsletter 우선 노출 힌트)
- `ref_event` 신규 추가 (국시 합격발표·간호법 등 시의성)

**의도적 데이터 품질 이슈 (참여자가 처리해야 할 과제)**
- **keywords_required 결측 1건**: id 12 (빈 셀) → topic·target·tone으로 현장 용어 추론·보완
- **ref_event 빈칸 다수**: id 02·04·08·09·13·14·18 (상시 콘텐츠 — 시즌 언급 없이 작성)
- **중복 topic 1건**: id 11 = id 17 ("실습 나가서 듣는 웃픈 한마디 모음") — target_audience 상이(간호학생 vs 경력RN)로, 같은 제목을 타깃별로 차별화해야 하는 함정

> **정답 컬럼 미포함 확인**: 채점 대상인 "완성 콘텐츠"는 데이터에 없음. keywords_required·platform_hint·ref_event는 노션이 입력값으로 명시한 hint성 컬럼이므로 포함.

## 4. Output 형식 결정

### Q. md 파일이면 안 됨 — 시각적으로 확인할 수 있어야 함

**결론**: 제출 본체는 `output_content_set.md`(노션 ⑤ 지정), 추가로 각 레벨별 HTML 샘플 생성

| 파일 | 주요 시각 요소 |
|------|---------------|
| `output/sample-basic.html` | 시드 카드 + 채널 탭(카드/카카오/뉴스레터) + 글자수 카운터 + 현장 용어 태그 |
| `output/sample-standard.html` | 파이프라인 플로우 + 콘텐츠 캘린더(주간 발행표) + 브랜드보이스 가이드 |
| `output/sample-challenge.html` | 우선순위 점수 계산기 + 발행 전략안 카드(P0/P1) + 캘린더 로드맵 |
